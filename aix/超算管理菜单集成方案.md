# AIX：超算管理菜单与 OpenSCOW 开关集成方案

本文档描述在 **AIX 工程**中如何根据 `config.yaml` 的 `openscow.enabled` 显示「超算管理」菜单，以及占位路由与服务端注入的实现约定。实现时应在 AIX 源码仓库中按本文档修改；OpenSCOW 本仓库中的 `aix/` 目录仅作方案归档。

---

## 1. 配置约定（`config.yaml`）

在 AIX 的 `config.yaml` 中增加：

```yaml
openscow:
  enabled: true   # false 或不写 openscow 块时不显示「超算管理」
```

**约定：**

- 未配置 `openscow` 或 `enabled` 缺省时，视为 **`enabled: false`**，避免误显示菜单。
- `enabled` 必须为布尔语义；若从 YAML 解析为字符串，需在服务端转为布尔后再注入前端。

---

## 2. 开关暴露给前端：服务端注入

**目标：** 首屏 HTML 中即带有开关状态，前端无需额外请求即可决定是否渲染菜单项（可与现有 SSR/布局模板一致）。

### 2.1 内联全局对象（示例）

在布局母版（如根 HTML 模板、`_document`、主布局组件对应的服务端输出）中，在合适位置（如 `</body>` 前）注入：

```html
<script>
  window.__AIX_RUNTIME__ = window.__AIX_RUNTIME__ || {};
  window.__AIX_RUNTIME__.openscow = { enabled: <%= openscowEnabled %> };
</script>
```

- 后端读取 `config.yaml`，将 `openscow.enabled` 解析为布尔变量 `openscowEnabled` 传入模板。
- 前端读取：

```javascript
const openscowEnabled = window.__AIX_RUNTIME__?.openscow?.enabled === true;
```

**注意：** 必须使用严格相等 `=== true`，避免字符串 `"true"` 等非布尔值误判。

### 2.2 可选：JSON Script 标签（利于 CSP）

```html
<script type="application/json" id="aix-runtime">
  {"openscow":{"enabled":true}}
</script>
```

前端：`JSON.parse(document.getElementById('aix-runtime').textContent)` 后取 `openscow.enabled`。

### 2.3 安全说明

若 `__AIX_RUNTIME__` 仅包含 `{ openscow: { enabled } }` 等非敏感开关，可直接暴露；勿将密钥、内网 URL 等一并写入全局对象。

---

## 3. UI：在「资源管理」右侧增加「超算管理」

**位置：** 在渲染导航的组件/菜单数据结构中，找到「资源管理」对应项，在其**之后**插入「超算管理」（数组顺序即横向「右侧」或侧栏「下一项」，具体以 AIX 导航布局为准）。

**显隐规则：**

- 仅当 **`openscow.enabled === true`** 时，将「超算管理」项加入菜单数据或渲染 VDOM。
- **`enabled !== true` 时：不要**把该项加入菜单数组，也**不要**用 `display: none` / `v-show="false"` 等方式隐藏已挂载的链接，避免 SEO 与无障碍（隐藏空链、焦点陷阱）问题。

---

## 4. 点击行为：占位路由（首期）

**不要求首期跳转真实业务地址**，避免使用 `href="#"` 或空的 `router.push` 引发异常或不良体验。

**做法：**

1. 新增路由，例如 **`/openscow`**（或项目统一前缀下的等价路径）。
2. 该页内容为占位文案即可，例如：**「即将接入 slurm」**（文案可按产品最终要求调整）。
3. 菜单项的 `to` / `path` / `href`（按所用路由框架写法）指向上述路径。
4. 后续接入 OpenSCOW / Slurm 时：优先修改该页面或在其下增加子路由、重定向，**导航入口路径可保持不变**。

**无障碍建议：** 占位页使用明确的标题（如 `<h1>`）与简短说明，避免读屏遇到完全空白页。

---

## 5. 测试与运维检查清单

| 检查项 | 说明 |
|--------|------|
| 默认关闭 | 未配置或 `enabled: false` 时，菜单中无「超算管理」 |
| 开启 | `enabled: true` 时，该项出现在「资源管理」之后 |
| 注入类型 | 前端判断使用 `=== true`，避免字符串真值问题 |
| 占位路由 | 可访问、无 `#` 空链、无未处理的路由错误 |
| 配置变更 | 若 AIX 支持热加载配置，需约定菜单是否随配置刷新；否则文档注明需重启 |

---

## 6. 与 OpenSCOW 的关系

- OpenSCOW 侧可有独立的 `mis.yaml` 等配置；AIX 的 `openscow.enabled` 仅控制 **AIX 控制台是否展示入口**，与 OpenSCOW 部署是否成功无自动耦合，由运维保证两边配置策略一致（可选）。
- 将来菜单子项或该页内 iframe/外链指向 MIS、Portal 等，在占位页或路由层扩展即可。
